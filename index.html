📐 核心公式
Precision / Recall / F1›
Precision = TP / (TP + FP)　被預測為正的，有多少真的是正
Recall = TP / (TP + FN)　真正的正，有多少被找出來
F1 = 2 × P × R / (P + R)　調和平均

醫療/詐欺偵測：Recall 優先（不能漏掉）
垃圾郵件：Precision 優先（不能誤刪重要信）
L1 vs L2 正則化›
L1 (Lasso)：λΣ|w|，部分權重歸零 → 稀疏模型、特徵選擇
L2 (Ridge)：λΣw²，所有權重縮小但不為零 → 防止整體過擬合

記憶：L1 = Lasso = Leave some zero
Bias-Variance 分解›
過擬合 (High Variance)：訓練好、測試差 → 加正則化、增資料、Dropout
欠擬合 (High Bias)：訓練測試都差 → 增加模型複雜度、特徵工程

學習曲線診斷：兩條線都高且接近 = 欠擬合
KNN / SVM / PCA 前處理›
KNN：距離型，必須 StandardScaler
SVM (RBF)：距離型，必須 StandardScaler
PCA：協方差型，必須 StandardScaler

決策樹 / 隨機森林 / XGBoost：不需要標準化
F1 計算範例›
TP=80, FP=20, FN=40
Precision = 80/100 = 0.80
Recall = 80/120 = 0.667
F1 = 2×0.80×0.667 / (0.80+0.667) = 0.727
Sklearn fit/transform›
fit()：從訓練資料學參數（只對 X_train）
transform()：套用已學參數轉換（X_train & X_test）
fit_transform()：等同 fit + transform，只用於訓練集

⚠️ 對 X_test 用 fit_transform = Data Leakage！
🧠 AI Pipeline 記憶串
資料收集 → 前處理（缺失值/標準化/編碼）→ 特徵工程 → 建模 → 訓練（Cross Validation）→ 評估（F1/AUC）→ 調校（Grid Search）→ 部署 → 監控（Data Drift）→ AI 治理

不平衡 → SMOTE → Recall → F1 → Stratified KFold（一組）
過擬合 → L1/L2 → Dropout → Early Stopping（一組）
